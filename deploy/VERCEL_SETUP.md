# Хостинг бота на Vercel (serverless)

Бот переписан под Vercel: вместо постоянно работающего процесса —
вебхук (`api/webhook.py`) и ежедневный Cron (`api/cron.py`). Данные учеников
(профили, попытки, статистика, активная сессия) хранятся в Upstash Redis (KV),
а банк заданий едет в бандле как read-only файл `letovo_bot/data/bank.sqlite`.

Рассылка идёт **раз в день в 7:00 по Москве** (04:00 UTC) — это задано в
`vercel.json` (`crons → "0 4 * * *"`). Время единое для всех учеников.

---

## Часть 1. Создать проект на Vercel

1. Зайдите на https://vercel.com и войдите через свой **GitHub**.
2. **Add New… → Project** → выберите репозиторий `rimmavadimovna-jpg/truetestbot`
   → **Import**.
3. Framework Preset оставьте **Other**. Ничего собирать не нужно —
   Vercel сам поставит зависимости из `requirements.txt`.
4. Пока **не** жмите Deploy — сначала добавим хранилище и переменные
   (можно и после; тогда после добавления переменных сделайте Redeploy).

## Часть 2. Подключить бесплатное KV-хранилище (Upstash Redis)

1. В проекте откройте вкладку **Storage → Create Database**.
2. Выберите **Upstash → Redis** (бесплатный тариф) → создайте.
3. На шаге **Connect** привяжите базу к этому проекту для окружений
   **Production, Preview, Development**.
   Vercel сам добавит переменные `KV_REST_API_URL` и `KV_REST_API_TOKEN`
   (бот понимает и их, и `UPSTASH_REDIS_REST_URL`/`_TOKEN`).

## Часть 3. Переменные окружения

**Settings → Environment Variables** (для Production). Добавьте:

| Имя | Значение | Обязательно |
|-----|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | токен бота от @BotFather | ✅ |
| `TELEGRAM_WEBHOOK_SECRET` | любая длинная случайная строка (придумайте) | ✅ |
| `CRON_SECRET` | любая длинная случайная строка (придумайте) | ✅ |
| `ANTHROPIC_API_KEY` | ключ Claude — включает уровень C проверки | опционально |

> `CRON_SECRET` Vercel автоматически подставляет в заголовок `Authorization`
> при вызове крона — отдельно ничего настраивать не надо.
> `KV_REST_API_*` уже добавлены на шаге 2.

## Часть 4. Деплой

- **Deployments → Deploy** (или просто `git push` в `main` — Vercel
  деплоит автоматически при каждом пуше).
- Дождитесь зелёного статуса. Запомните адрес продакшена, например
  `https://truetestbot.vercel.app`.
- Проверка: откройте `https://<ваш-домен>/api/webhook` в браузере —
  должно ответить `letovo-bot webhook is up`.

## Часть 5. Привязать вебхук Telegram (однократно)

Выполните один запрос (подставьте токен, домен и тот же секрет, что в
`TELEGRAM_WEBHOOK_SECRET`). Можно прямо в браузере:

```
https://api.telegram.org/bot<ТОКЕН>/setWebhook?url=https://<ваш-домен>/api/webhook&secret_token=<TELEGRAM_WEBHOOK_SECRET>
```

Ответ `{"ok":true,"result":true,...}` — готово. Проверить статус:

```
https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo
```

После этого напишите боту `/start` в Telegram — он должен ответить.
Ежедневный набор будет приходить автоматически в 7:00 МСК; получить вопросы
сразу — команда `/today`.

## Обновления в будущем

Любой `git push` в ветку `main` → Vercel пересобирает и деплоит автоматически.
Если поменяли задания — пересоберите банк локально и закоммитьте его:

```
python -m letovo_bot.data.build_bank
git add letovo_bot/data/bank.sqlite && git commit -m "rebuild bank" && git push
```

## Заметки и ограничения

- **Тариф Hobby (бесплатный)**: один ежедневный Cron — нам ровно подходит
  (рассылка раз в сутки). Возможна задержка срабатывания на несколько десятков
  минут — для учебной рассылки некритично.
- **Персональное время рассылки отключено**: время фиксировано (7:00 МСК),
  команда `/settings` теперь только показывает расписание.
- **Локальный запуск** для отладки по-прежнему работает через polling:
  `python -m letovo_bot.bot.main` (без переменных Upstash данные живут в памяти
  процесса).
- Альтернатива Vercel — постоянно работающий сервер (см. `SERVER_SETUP.md`):
  там код проще (ничего переписывать не надо), но нужен always-on хост.
