# Letovo Bot — тренажёр для вступительного теста (русский язык, 7 класс)

Telegram-бот, который каждый день присылает ученику небольшой набор заданий
(15–20 минут) строго в формате вступительного теста Летово, проверяет ответы
автоматически, даёт обучающую обратную связь и ведёт статистику по темам.

Реализованы **12 автоматизируемых типов** заданий (1–12). Задания 13 (сжатый
пересказ) и 14 (сочинение) исключены — это продуктивная часть без объективной
автопроверки.

## Главный принцип: никакой генерации языкового материала «на лету»

Языковая модель **не придумывает** слова, орфограммы, предложения,
транскрипции и эталоны. Архитектура из четырёх слоёв:

1. **Слой данных** (`data/bank.sqlite`) — выверенный банк. Каждая единица
   размечена, хранит правильный ответ, тип орфограммы/признак и ссылку на
   справочник (`source`).
2. **Слой сборки** (`core/assembler.py`) — детерминированный Python собирает
   задание и проверяет инварианты (`assert`) перед выдачей.
3. **Слой проверки** (`core/checker.py`, `core/detectors.py`, `core/llm.py`) —
   три уровня: детекторы → сверка с эталоном → LLM-судья (только остаточные
   субъективные критерии, по строгой рубрике и эталону).
4. **LLM** используется только для (а) разовой офлайн-выверки банка и
   (б) проверки остаточных критериев открытых ответов. При обычной выдаче
   обращений к LLM нет.

## Установка

```bash
pip install -r requirements.txt
```

Python 3.11+. Ключевые зависимости: `aiogram` 3.x, `anthropic`, `pydantic`,
`upstash-redis` (KV-хранилище учеников на serverless), `pymorphy3` (морфология),
`razdel` (токенизация).

## Хостинг

- **Vercel (serverless)** — рекомендуемый способ, см. `deploy/VERCEL_SETUP.md`.
  Вебхук (`api/webhook.py`) + ежедневный Cron (`api/cron.py`, 10:00 МСК) +
  Upstash Redis для данных учеников; банк едет read-only файлом в бандле.
- **Always-on сервер** (Oracle Cloud Free и т.п.) — см. `deploy/SERVER_SETUP.md`.

## Переменные окружения

| Переменная             | Назначение                                              |
|------------------------|---------------------------------------------------------|
| `TELEGRAM_BOT_TOKEN`   | **обязательно** — токен бота                            |
| `ANTHROPIC_API_KEY`    | опционально — включает уровень C и офлайн-выверку       |
| `ANTHROPIC_MODEL_VERIFY` | модель для офлайн-выверки банка (по умолч. сильная)   |
| `ANTHROPIC_MODEL_JUDGE`  | модель для рантайм-проверки (по умолч. быстрая)       |
| `LETOVO_BANK_PATH`     | путь к `bank.sqlite`                                    |
| `LETOVO_DEFAULT_TZ`    | часовой пояс по умолчанию (`Europe/Podgorica`)          |
| `LETOVO_DEFAULT_TIME`  | время рассылки по умолчанию (`18:00`)                   |

Секреты берутся **только** из окружения. Актуальные строки моделей сверяй на
docs.claude.com.

## Наполнение и выверка банка

```bash
# 1. Собрать банк из выверенного семени (seed) и проверить инварианты
python -m letovo_bot.data.build_bank

# 2. (опционально) Офлайн-выверка орфограмм через Claude API
python -c "from letovo_bot.core import db; from letovo_bot.config import BANK_PATH; \
from letovo_bot.data.verify import verify_orthograms; \
c=db.connect(BANK_PATH); print('verified:', verify_orthograms(c))"

# 3. (опционально) Выверка словообразовательных цепочек зад. 9 по Викисловарю
python -c "from letovo_bot.core import db; from letovo_bot.config import BANK_PATH; \
from letovo_bot.data.verify import verify_chains; \
c=db.connect(BANK_PATH); print('confirmed:', verify_chains(c))"

# 4. (опционально) Сверка чисел зад. 7 с офлайн-транскрипцией согласных
python -c "from letovo_bot.core import db; from letovo_bot.config import BANK_PATH; \
from letovo_bot.data.verify import verify_phonetics; \
c=db.connect(BANK_PATH); print('agreed:', verify_phonetics(c))"
```

- В банке несколько выверенных вариантов на каждый из 12 типов (для ротации и
  адаптивного подбора) и два собственных текста для зад. 11–12.
- `verify_orthograms` помечает `verified=1` только при `confident=true`;
  остальное уходит в `for_manual_review.csv`.
- **Зад. 7** (`data/phonetics.py`) — приближённая школьная транскрипция
  согласных (оглушение/озвончение, мягкость). Это инструмент ВЫВЕРКИ, не
  источник истины: число хранится в банке, а `verify_phonetics` лишь сигналит
  о расхождениях в `for_manual_review_7.csv`.
- **Зад. 9** выверяется по Викисловарю через MediaWiki API (`action=parse`,
  без парсинга HTML). Шаг цепочки `A → B` включается только если статья `B`
  явно подтверждает производность от `A` («происходит от», «образовано от»);
  иначе шаг бракуется и уходит на ручную проверку.
- **Авторские права**: тексты Летово в банк не копируются. Для зад. 11–12
  используются собственные/свободно лицензированные тексты с самостоятельно
  составленными утверждениями и ключами. Форматы заданий не охраняются.

Чтобы добавить материал — расширь `seed_*`-функции в `data/build_bank.py`
(или импортируй из `data/sources/`) и пересобери банк. Сборка падает при
нарушении любого инварианта.

## Запуск бота

```bash
export TELEGRAM_BOT_TOKEN=...           # обязательно
export ANTHROPIC_API_KEY=...            # опционально (уровень C)
python -m letovo_bot.data.build_bank    # если банк ещё не собран
python -m letovo_bot.bot.main
```

### Команды бота

- `/start` — регистрация, часовой пояс, время рассылки.
- `/today` — получить набор заданий прямо сейчас.
- `/stats` — прогресс по темам (от слабых к сильным).
- `/theory <тема>` — правила и ссылки на источники.
- `/settings [time HH:MM | tz Zone]` — настройки рассылки.

Задания с выбором номеров (зад. 2, 6, 11) — inline-кнопки-тогглы + «Готово»;
остальные — текстом. После каждого ответа — мгновенная проверка: вердикт,
разбор по критериям, эталонный/образцовый ответ и ссылка на правило.

## Конвейер проверки (§3)

- **Уровень A** (`detectors.py`) — детерминированные детекторы: пунктуация,
  грамматические основы, схемы, использование заданных слов/фразеологизмов,
  различие корней, множества номеров, числа. Без LLM.
- **Уровень B** (`checker.py`) — сверка с эталоном/списком допустимых вариантов
  из банка. Совпало с допустимым — засчитано без LLM.
- **Уровень C** (`llm.py`) — LLM-судья только для остаточных субъективных
  критериев, всегда с эталоном и строгой рубрикой. Не имеет права засчитать то,
  чего нет в банке (для объективных частей). При невалидном JSON — ответ
  помечается «нужна ручная проверка», а не угадывается.

Без `ANTHROPIC_API_KEY` уровень C отключён; остаточные субъективные вопросы
помечаются `needs_review` и складываются в лог для преподавателя.

## Тесты

```bash
python -m pytest letovo_bot/tests/ -q
```

- `test_invariants.py` — инварианты банка (по одному «лишнему» в ряду зад. 1/10,
  непустой однозначный эталон, число зад. 7 хранится, шаги зад. 9 со ссылками
  Викисловаря, тексты 11–12 не из Летово).
- `test_detectors.py` — детекторы уровня A.
- `test_checker.py` — эталонные ответы получают полный балл, неверные — низкий.
- `test_assembler.py` — валидация банка и адаптивный подбор набора.

CI (`.github/workflows/ci.yml`) гоняет тесты и пересборку банка — падает при
нарушении инвариантов.
